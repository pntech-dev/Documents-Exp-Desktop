## 📌 Обзор

**Documents Exp** разработано для оптимизации управления технической документацией в организации. Приложение предоставляет структурированное представление документов по отделам и категориям, а также обеспечивает эффективный поиск и редактирование.

Приложение работает в двух режимах:

1.  **Режим Гостя:** Только просмотр и поиск документов.
2.  **Режим Пользователя:** Полный доступ к созданию, редактированию, удалению, импорту и экспорту.

## 🎯 Ключевые возможности

### ✅ Управление документами

- Организация по **Отделам** и **Категориям**.
- **Бесконечная прокрутка** (Pagination) для больших списков.
- **Поиск** с фильтрацией.

### ✅ Редактор документов

- Создание и редактирование документов.
- Управление страницами: **Добавление**, **Дублирование**, **Удаление**.
- Перетаскивание страниц (Drag-and-drop) для изменения порядка.

### ✅ Импорт и Экспорт

- **Импорт** страниц из файлов `.docx` (Word).
- **Экспорт** документов в формат `.docx`.
- **Печать** документов напрямую из приложения.

### ✅ UI/UX

- Современный интерфейс на **PyQt5**.
- **Тёмная** и **Светлая** темы оформления.
- Адаптивная боковая панель и навигация.
- Всплывающие уведомления (Toast notifications).

### ✅ Безопасность

- Безопасное хранение токенов с использованием системного **Keyring**.
- Функция автоматического входа.
- Восстановление пароля через email.

## 🏗 Архитектура

Проект следует паттерну **MVC (Model-View-Controller)** для разделения логики и интерфейса.

```
┌──────┐     ┌────────────┐     ┌───────┐
│ View │ <-- │ Controller │ --> │ Model │
└──────┘     └────────────┘     └───────┘
  UI           Логика           Данные/API
```

- **View (Представление):** Отвечает за отрисовку UI и события пользователя.
- **Controller (Контроллер):** Управляет логикой, обрабатывает сигналы и связывает View и Model.
- **Model (Модель):** Управляет данными, общением с API и бизнес-логикой.

## 🔧 Установка

1. Клонируйте репозиторий:

    ```bash
    git clone https://github.com/your-repo/documents-exp-desktop.git
    cd documents-exp-desktop
    ```

2. Установите зависимости:

    ```bash
    pip install -r requirements.txt
    ```

3. Настройка:
   Отредактируйте `config.yaml`, указав URL вашего API:

    ```yaml
    base_url: "http://127.0.0.1:8000"
    ```

4. Запуск:
    ```bash
    python app.py
    ```
